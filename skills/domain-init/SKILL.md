# domain-init

Generate or update `README.md` at the repo root from project scan and optional input documents.

## Arguments

- `$1` (optional): path to input documents (spec, PRD, design doc, API contract).
  Can be a single file, a directory, or a glob pattern.
  If omitted, generate entirely from project structure scan.

---

## Procedure

### Step 1 — Collect input documents (if `$1` provided)

Read all files at the path provided. From them, extract:
- **Business entities** — nouns that appear as domain objects (User, Order, Product, Invoice…)
- **Core workflows** — verb phrases describing key operations (create order, authenticate user…)
- **External systems** — databases, queues, third-party APIs mentioned
- **Bounded contexts** — distinct domain areas or subdomains referenced

Record these as a structured list for use in Step 4.

### Step 3 — Scan project structure

Run the following to understand the codebase:

```bash
# Top-level structure
find . -maxdepth 3 -type d -not -path '*/.git/*' -not -path '*/vendor/*' \
       -not -path '*/node_modules/*' | sort

# Entry points (Go)
grep -rl "^package main" --include="*.go" . 2>/dev/null | head -20

# Key dependencies (Go)
cat go.mod 2>/dev/null || true

# Key dependencies (Node)
cat package.json 2>/dev/null || true

# Existing conventions
cat CLAUDE.md 2>/dev/null || true
```

From the scan, identify:
- Top-level packages and their purpose (infer from directory names and a quick file sample)
- Entry point binaries/services
- External dependencies (DB drivers, HTTP frameworks, queue clients, gRPC)
- Existing conventions from `CLAUDE.md` if present

### Step 4 — Handle existing file (idempotency)

Check if `README.md` already exists at the repo root.

**If it exists:**
1. Read the current file.
2. Identify any blocks marked with `<!-- MANUAL -->` … `<!-- /MANUAL -->` — these are
   hand-written sections that must not be overwritten.
3. Extract and preserve their content verbatim.
4. Regenerate all non-MANUAL sections with fresh data from Steps 2–3.
5. After writing, output a diff summary: which sections changed, which were preserved.

**If it does not exist:**
Generate the full file from scratch.

### Step 5 — Write `README.md`

Write the file at the repo root with this exact structure. Use the data collected in Steps 2–3.
Preserve any `<!-- MANUAL -->` blocks extracted in Step 4.

```markdown

> Domain knowledge auto-generated by domain-init. Preserve <!-- MANUAL --> blocks for hand-written notes.
> Re-run `/domain-init` after major structural changes; hand-written blocks are safe.
```

### Step 6 — Report output

After writing the file, output:

```

  Written: README.md (repo root)
  Sections generated: [list]
  MANUAL blocks preserved: [count] ([list of section names])
  Items needing manual review: [list — e.g., entities that couldn't be auto-detected]

Next step: invoke the `domain` skill or run `@feature` — domain context is now ready.
```
